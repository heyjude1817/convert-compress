## P1

### 4. 完成文件夹监控输出目录配置 UI
- 优先级：P1
- 状态：已完成
- 任务：补齐文件夹监控设置中的输出目录配置
- 当前状态：`FolderMonitorManager` 和 `MonitoredFolder` 已支持 `outputURL` / `outputBookmarkData`
- 缺口：设置页仅支持目录、启用开关、Preset，下游没有输出目录选择入口
- 执行项：在文件夹监控列表中增加输出目录展示与选择；保存 security-scoped bookmark；允许清空回退到默认输出目录
- 验收标准：可为单个监控目录配置输出目录；重启后仍可恢复；自动处理写入目标目录

### 5. 让批量重命名冲突处理真正生效
- 优先级：P1
- 状态：已完成
- 任务：把命名模板的批次冲突消解接入实际导出流程
- 当前状态：`NamingTemplateResolver.resolveBatch()` 已实现
- 缺口：`ProcessingPipeline.run(on:)` / `plannedDestinationURL(for:)` 未传入真实批次序号，也未在批量导出时统一消解重名
- 执行项：在批量导出阶段先生成整批文件名计划；对同批次冲突追加 `_1`、`_2`；UI 预估和实际导出使用同一套逻辑
- 验收标准：同模板导出多张图片且结果重名时，输出文件名稳定且无覆盖

### 6. 补齐错误处理规格缺口
- 优先级：P1
- 状态：已完成
- 任务：让逐图错误处理达到原规划行为
- 当前状态：ViewModel 已收集 `ProcessingError` 并在导出完成后展示
- 缺口：`ProcessingError` 缺少 `.insufficientDisk`；`writeFailed` 未完整接入；`Pipeline` 仍为 `throws` 模式但缺少对应测试保障
- 执行项：补充错误类型与映射；明确写入失败、权限失败、磁盘空间不足的用户可见文案；为当前实现补测试
- 验收标准：单张失败不影响整批；错误原因可区分；测试覆盖关键异常路径
