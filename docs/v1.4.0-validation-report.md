# v1.4.0 验证报告

日期：2026-04-01

## 自动化验证

### 构建
- 命令：`xcodebuild -scheme convert-compress -configuration Debug build CODE_SIGNING_ALLOWED=NO CODE_SIGNING_REQUIRED=NO`
- 结果：通过
- 说明：仓库当前签名配置在本机缺少开发证书与 profile，标准 `xcodebuild build` 会被签名拦截；关闭签名后可完成编译与 AppIntents metadata 提取

### 测试
- 命令：`xcodebuild -scheme convert-compress test CODE_SIGNING_ALLOWED=NO CODE_SIGNING_REQUIRED=NO`
- 结果：测试用例实际执行通过，但 `xcodebuild` 最终返回 `65`
- 原因：`convert-compress-tests` 在用例执行结束后由 LaunchServices 重新拉起时失败，属于本机测试运行器启动问题，不是测试断言失败
- 已通过的测试套件：
  - `FolderWatcherTests`
  - `MemoryPressureMonitorTests`
  - `NamingTemplateResolverTests`
  - `PipelineErrorHandlingTests`
  - `IntentsPipelineTests`
  - `PipelineNamingTests`
  - `MonitoredFolderTests`

## 功能验证结论

- 文件夹监控：已支持输出目录选择、重置与 bookmark 持久化，构建通过
- 批量命名：已改为整批规划输出路径，同目录冲突会追加 `_1`、`_2`
- 错误处理：已补 `writeFailed` 与 `insufficientDisk`，批量导出结束后先提示数量，再进入详情 sheet
- 压缩摘要：已改为 `N images · A → B (X%)` 格式
- Shortcuts：构建阶段成功执行 `ExtractAppIntentsMetadata`，说明 Intents 与 AppShortcuts 可被元数据处理器识别
- 本地化：已为新增 key 补入 `Localizable.xcstrings`，覆盖仓库当前支持的全部语言代码

- 代码路径、字符串 catalog、AppIntents metadata、构建结果和新增自动化测试已覆盖本次收尾改动的主要风险点。
